🏗️ Model-driven Apps là gì?

Model-driven apps là loại ứng dụng được xây dựng trên nền tảng Dataverse (trước đây gọi là Common Data Service). Thay vì thiết kế từng pixel như Canvas Apps, bạn xây dựng dựa trên mô hình dữ liệu — layout được tự động tạo dựa trên components bạn thêm vào.

💡 Khác biệt cốt lõi
Canvas Apps = "Bắt đầu từ UI" — bạn thiết kế giao diện trước, kết nối dữ liệu sau.
Model-driven Apps = "Bắt đầu từ Data" — bạn định nghĩa dữ liệu trước, giao diện tự động được tạo.
📊

Data-First Approach

Thiết kế tables, columns, relationships trong Dataverse trước. App tự động phản ánh structure.

🧩

Component-Based

Thêm forms, views, charts, dashboards. Layout tự động responsive và nhất quán.

📱

Responsive UI

Giao diện tự động điều chỉnh cho desktop, tablet, mobile — không cần thiết kế riêng.

⚙️

Business Logic

Business rules, workflows, Power Automate flows tích hợp sẵn trong model.

🧩 Các thành phần chính

Model-driven app được cấu thành từ các components cốt lõi:

📋

Forms

Biểu mẫu nhập/xem dữ liệu. Main, Quick Create, Quick View forms.

📊

Views

Danh sách records với cột, filter, sort tùy chỉnh.

📈

Charts

Biểu đồ trực quan cho dữ liệu: bar, pie, line, funnel.

📰

Dashboards

Tổng hợp views, charts, web resources trên một trang.

📐

Business Rules

Logic nghiệp vụ: validation, show/hide, set values, locks.

🗺️

Site Map

Cấu trúc navigation: Areas, Groups, Sub Areas.

🚀 Tạo Model-driven App

Tạo Solution trong Dataverse

Vào make.powerapps.com → Solutions → New solution. Solution chứa tất cả components của app.

Tạo / Import Tables

Thêm tables (entities) cần thiết: custom tables hoặc standard tables (Account, Contact...).

Thiết kế Forms & Views

Tùy chỉnh Main Form (thêm/sắp xếp columns, tabs, sections) và Views (chọn cột hiển thị, filter).

Tạo Model-driven App

New → App → Model-driven app. Sử dụng app designer để thêm tables, forms, views.

Cấu hình Site Map

Thiết kế navigation: tạo Areas, Groups, Sub Areas dẫn đến tables/views/dashboards.

Validate & Publish

Kiểm tra app, sửa lỗi (nếu có), publish to make live.

📋 Forms — Biểu mẫu dữ liệu

Forms là giao diện chính để người dùng tương tác với records. Có nhiều loại form:

Loại FormMô tảKhi nào dùng
Main FormForm đầy đủ với tabs, sections, fieldsXem/chỉnh sửa record chi tiết
Quick CreateForm popup nhỏ, ít fieldsTạo record mới nhanh
Quick ViewForm read-only, nhúng trong form khácXem dữ liệu liên quan
Card FormForm dạng card cho viewsHiển thị trong danh sách

Cấu trúc Main Form

  • Header: Hiển thị fields quan trọng nhất (luôn visible)
  • Tabs: Chia form thành các nhóm nội dung (General, Details, Notes...)
  • Sections: Mỗi tab chứa một hoặc nhiều sections
  • Fields: Columns từ table, hiển thị theo loại dữ liệu
  • Sub-grids: Danh sách related records (1:N relationships)
  • Timeline: Lịch sử activities (emails, calls, notes)

📊 Views — Giao diện danh sách

Views hiển thị danh sách records theo cấu hình tùy chỉnh:

Loại ViewMô tả
Public ViewsMọi user đều thấy. Ví dụ: "Active Contacts"
Personal ViewsChỉ người tạo thấy
System ViewsViews hệ thống: "My Records", "All Records"
Quick Find ViewView cho thanh tìm kiếm
Lookup ViewView cho lookup dialogs

Mỗi view cho phép bạn cấu hình: columns hiển thị, sort order, filter criteria, và column widths.

📈 Charts & Dashboards

Charts

Các loại chart hỗ trợ: Bar, Column, Pie, Line, Funnel, Tag, Doughnut. Charts kết nối trực tiếp với views và cập nhật real-time.

Dashboards

Dashboard tổng hợp nhiều components:

  • Charts: Biểu đồ trực quan
  • Lists (Views): Danh sách records
  • Web Resources: HTML, JavaScript tùy chỉnh
  • IFRAMEs: Nhúng trang web bên ngoài
  • Power BI tiles: Nhúng Power BI reports
💡 Interactive Dashboards
Sử dụng Interactive dashboards cho trải nghiệm người dùng tốt hơn — cho phép filter trực tiếp bằng cách click vào chart segments, và visual filters hiển thị real-time.

📐 Business Rules

Business Rules cho phép bạn thêm logic nghiệp vụ mà không cần code:

ActionMô tảVí dụ
Show ErrorHiện thông báo lỗiEmail format không đúng
Set ValueGán giá trị cho fieldAuto-fill city từ zip code
Set VisibilityẨn/hiện fieldẨn "Spouse" nếu marital = Single
Lock/UnlockKhóa/mở khóa fieldLock fields khi status = Closed
Set RequiredBắt buộc/optionalPhone required nếu method = Call
RecommendationĐề xuất giá trị (AI)Suggest category cho case
Ví dụ Business Rule Logic
IF Condition:
    Field: "Priority" Equals "High"
THEN Action:
    Set Required: "Resolution Date" = Required
    Show Error: "High priority cases must have resolution date"
    Set Value: "SLA Days" = 3

🗺️ Site Map Designer

Site Map định nghĩa cấu trúc navigation của model-driven app:

  • Area: Phần lớn nhất (ví dụ: Sales, Service, Marketing)
  • Group: Nhóm trong Area (ví dụ: Customers, Products)
  • Sub Area: Mục cụ thể dẫn đến table/view/dashboard/URL
Ví dụ cấu trúc Site Map
📁 Area: Sales
  📂 Group: Customers
    📄 Sub Area: Contacts → Contact table
    📄 Sub Area: Accounts → Account table
  📂 Group: Orders
    📄 Sub Area: Orders → Order table
    📄 Sub Area: Products → Product table
📁 Area: Service
  📂 Group: Cases
    📄 Sub Area: Active Cases → Case (filter: Active)
    📄 Sub Area: Dashboard → Service Dashboard

✅ Validate & Publish

Validate

Click "Validate" trong app designer. Hệ thống kiểm tra tất cả components và hiển thị warnings/errors.

Fix Issues

Sửa các lỗi: thiếu form, view chưa được thêm, site map invalid, etc.

Save & Publish

Save app, sau đó click "Publish". App trở thành live cho tất cả users được assigned.

Play / Test

Click "Play" để test app trong browser. Kiểm tra navigation, forms, views, business rules.
