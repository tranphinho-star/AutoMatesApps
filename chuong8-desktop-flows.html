🤖 Desktop Flows & RPA là gì?

RPA (Robotic Process Automation) — tự động hóa các tác vụ lặp đi lặp lại trên desktop: click buttons, nhập dữ liệu, điều khiển ứng dụng, scrape web... giống như một "robot" thao tác thay bạn.

🖱️

UI Automation

Click, type, select trên bất kỳ ứng dụng desktop/web app nào.

📊

Excel Automation

Mở, đọc, ghi, format Excel files — không cần viết VBA.

🌐

Web Scraping

Trích xuất dữ liệu từ websites, fill forms, navigate pages.

📁

File Operations

Copy, move, rename, compress files & folders tự động.

📝 Attended vs Unattended
Attended: Người dùng đang trước màn hình, flow chạy cùng lúc.
Unattended: Flow chạy trên VM/server khi không ai đang dùng. Yêu cầu license riêng.

📥 Cài đặt & Setup

Cài đặt Power Automate for Desktop

Tải từ flow.microsoft.com hoặc Microsoft Store. Miễn phí cho Windows 10/11 (bản cơ bản).

Cài Browser Extension

Cài extension cho Chrome/Edge/Firefox để hỗ trợ web automation và UI element recognition.

Đăng nhập

Sử dụng tài khoản Microsoft (personal hoặc work/school). Work account cho full features.

Tạo Dataverse Database (Optional)

Cần Dataverse environment để lưu desktop flows lên cloud và chia sẻ.

🖥️ Power Automate Console

Console là cửa sổ chính khi mở Power Automate for Desktop:

📋

Danh sách desktop flows. Tạo mới, edit, run, share.

📊

Examples

Flows mẫu để học và tham khảo.

⚙️

Settings

Cấu hình: default env, notifications, machine settings.

📝

Xem lịch sử chạy, errors, execution details.

🎨 Desktop Flow Designer

Designer là IDE để tạo và chỉnh sửa desktop flows:

  • Actions pane (trái): 400+ actions được phân nhóm theo category
  • Workspace (giữa): Nơi kéo thả và sắp xếp actions thành flow
  • Variables pane (phải): Quản lý input/output variables, flow variables
  • Subflows: Chia flow lớn thành các subflows nhỏ, tái sử dụng
  • UI Elements: Panel hiển thị captured UI elements cho automation
  • Errors pane: Hiển thị warnings, errors khi design time

Recorder

Sử dụng Desktop Recorder hoặc Web Recorder để record các thao tác, tự động tạo actions tương ứng. Chỉnh sửa sau khi record.

💡 Variable naming
Sử dụng tên biến mô tả rõ ràng: %CustomerName%, %TotalAmount%, %FilesList%. Desktop flows dùng %VariableName% syntax.

🌐 Web & UI Automation

Web Automation

Ví dụ: Web Scraping
1. Launch new Chrome → URL: "https://example.com/reports"
2. Click element → Login button
3. Populate text field → Username: %Username%
4. Populate text field → Password: %Password%
5. Click element → Submit button
6. Wait for page → Contains "Dashboard"
7. Extract data from web page → Table element → %ReportData%
8. Write to Excel → %ReportData% → Sheet1
9. Close browser

UI Automation (Desktop Apps)

Ví dụ: Nhập dữ liệu vào ERP
1. Run application → "C:\Program Files\ERP\erp.exe"
2. Wait for window → Title: "ERP System"
3. Focus window → "ERP System"
4. FOR EACH row IN %ExcelData%
   4.1. Click element → "New Record" button
   4.2. Populate text field → Name: %row['Name']%
   4.3. Populate text field → Amount: %row['Amount']%
   4.4. Click element → "Save" button
   4.5. Wait → 1 second
5. END FOR

🖥️ Machines & Machine Groups

Machines là các computer được đăng ký để chạy desktop flows:

  • Register machine: Đăng ký máy tính với Power Automate cloud
  • Machine groups: Nhóm nhiều machines — load balancing, high availability
  • Maintenance mode: Tạm dừng machine khỏi group để bảo trì
  • Connection credentials: Cấu hình Windows login credentials cho unattended runs

☁️ Trigger từ Cloud Flow

Kết hợp Cloud Flow + Desktop Flow tạo hybrid automation mạnh mẽ:

Hybrid Flow — Cloud triggers Desktop
CLOUD FLOW:
  TRIGGER: When a new invoice arrives (email)
    ↓
  ACTION: Get attachment content
    ↓
  ACTION: Run a flow built with PA Desktop
    → Machine: "AccountingPC"
    → Desktop flow: "Enter Invoice to SAP"
    → Input: InvoiceData, AttachmentContent
    ↓
  ACTION: Send confirmation email

📊 Monitor & Troubleshoot

  • Run logs: Chi tiết từng action: success/fail, duration, screenshots
  • Desktop flow activity: Xem từ Power Automate portal
  • Analytics: Success rate, average run time, error trends
  • DLP policies: Data Loss Prevention cho desktop flows
⚠️ Tips Troubleshooting
  • Thêm Wait actions giữa UI clicks (đợi UI load)
  • Sử dụng Error Handling (On block error) cho mỗi action quan trọng
  • Dùng Screenshot actions để debug visual issues
  • Kiểm tra screen resolution trên machine phải giống lúc design
