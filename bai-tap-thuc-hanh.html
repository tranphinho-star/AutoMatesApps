⭐ BT1: Canvas App từ SharePoint List

📝 Mức độ: Cơ bản | Thời gian: 15 phút | Chương liên quan: Ch.2

Mục tiêu

Tạo Canvas App tự động từ SharePoint list — hiểu cấu trúc 3-screen app cơ bản.

Yêu cầu

  1. Tạo SharePoint list "Employees" với các columns: Name (Text), Email (Text), Department (Choice: IT, HR, Sales, Marketing), Phone (Text), Start Date (Date)
  2. Thêm 5-10 records mẫu
  3. Tạo Canvas App từ SharePoint list (Start from data)
  4. Test 3 screens: Browse, Detail, Edit
  5. Publish và chia sẻ app

Hướng dẫn chi tiết

Tạo SharePoint List

Truy cập SharePoint site → New → List → Đặt tên "Employees" → Thêm columns theo yêu cầu. Lưu ý: column "Department" là Choice type.

Tạo App

make.powerapps.com → Create → Start from data → SharePoint → Chọn site → Chọn list "Employees". Đợi app được tạo tự động.

Khám phá 3 screens

Xem cấu trúc: BrowseScreen1 (Gallery), DetailScreen1 (Display form), EditScreen1 (Edit form). Nhấn F5 để preview.

Tùy chỉnh

Thay đổi Gallery layout: hiển thị Name + Department. Thêm icon ở Detail screen. Đổi màu header.

Save & Publish

Ctrl+S → Save → File → Publish. Test trên mobile bằng Power Apps mobile app.

⭐⭐ BT2: Canvas CRUD App từ Blank

📝 Mức độ: Trung bình | Thời gian: 30 phút | Chương liên quan: Ch.2

Mục tiêu

Tự tay tạo Canvas App từ blank với đầy đủ CRUD operations.

Yêu cầu

  1. Tạo app blank (Phone layout)
  2. Kết nối tới data source (SharePoint list hoặc Excel)
  3. Tạo Home Screen với gallery hiển thị danh sách products
  4. Tạo Add Screen với form thêm product mới (Name, Price, Category, InStock)
  5. Tạo Edit Screen với form chỉnh sửa product
  6. Thêm nút Delete với xác nhận (Confirm dialog)
  7. Thêm navigation giữa các screens

Power Fx cần sử dụng

Power Fx Functions
// Navigate — Chuyển screen
Navigate(AddScreen, ScreenTransition.Fade)

// Patch — Tạo record mới
Patch(Products, Defaults(Products), {
    Name: txtName.Text,
    Price: Value(txtPrice.Text),
    Category: ddCategory.Selected.Value,
    InStock: tglInStock.Value
})

// Patch — Cập nhật record
Patch(Products, Gallery1.Selected, {
    Name: txtEditName.Text
})

// Remove — Xóa record
Remove(Products, Gallery1.Selected)

// Notify — Thông báo
Notify("Product saved!", NotificationType.Success)

// Back — Quay lại screen trước
Back()

⭐⭐ BT3: Gallery + Search + Filter nâng cao

📝 Mức độ: Trung bình | Thời gian: 25 phút | Chương liên quan: Ch.2

Mục tiêu

Xây dựng gallery với search, filter, sort — hiểu Power Fx nâng cao.

Yêu cầu

  1. Hiển thị gallery với hình ảnh, tên, giá, và category
  2. Search box tìm kiếm theo tên sản phẩm
  3. Dropdown filter theo Category
  4. Sort button: A-Z / Z-A / Price Low-High / Price High-Low
  5. Hiển thị tổng số kết quả tìm được
  6. Nút "Clear Filters" reset tất cả

Formula mẫu

Gallery Items Property
SortByColumns(
    Filter(
        Search(
            Products,
            txtSearch.Text,
            "Name"
        ),
        Or(
            ddCategory.Selected.Value = "All",
            Category = ddCategory.Selected.Value
        )
    ),
    ddSort.Selected.Column,
    If(ddSort.Selected.Order = "Asc",
       SortOrder.Ascending,
       SortOrder.Descending
    )
)

⭐⭐⭐ BT4: Model-driven CRM App

📝 Mức độ: Nâng cao | Thời gian: 45 phút | Chương liên quan: Ch.3, Ch.4

Mục tiêu

Xây dựng mini CRM (Customer Relationship Management) app với Model-driven approach.

Yêu cầu

  1. Tạo 3 custom tables trong Dataverse:
    • Customers: Name, Email, Phone, Company, Status (Active/Inactive)
    • Deals: Title, Amount (Currency), Stage (Choice: Lead/Qualified/Proposal/Won/Lost), CloseDate
    • Activities: Subject, Description, Type (Call/Email/Meeting), Date
  2. Thiết lập relationships: Customer → Deals (1:N), Customer → Activities (1:N)
  3. Tạo forms: Customize Main Form cho mỗi table
  4. Tạo views: "Active Customers", "Open Deals", "This Week Activities"
  5. Tạo Business Rule: Khi Deal Stage = "Won" → Lock Amount field
  6. Tạo Model-driven app với site map: Sales → Customers, Deals | Activities → All Activities
  7. Tạo Dashboard: "Sales Overview" với chart Deals by Stage + list Open Deals

⭐ BT5: Automated Cloud Flow — Email Notification

📝 Mức độ: Cơ bản | Thời gian: 10 phút | Chương liên quan: Ch.7

Mục tiêu

Tạo automated flow: gửi email thông báo khi có item mới trên SharePoint.

Yêu cầu

  1. Trigger: "When an item is created" trên SharePoint list
  2. Action 1: Format email body với HTML template (tên, ngày tạo, link tới item)
  3. Action 2: Gửi email tới manager
  4. Action 3: Post message lên Teams channel
  5. Test bằng cách thêm item mới vào SharePoint list

Hướng dẫn

Tạo Flow

make.powerautomate.com → Create → Automated cloud flow → Đặt tên → Chọn trigger "When an item is created (SharePoint)".

Cấu hình Trigger

Chọn Site Address và List Name.

Thêm Actions

+ New step → "Send an email (V2)" → Cấu hình To, Subject, Body. Sử dụng dynamic content từ trigger.

Test

Click Test → Manually → Thêm item vào SharePoint → Kiểm tra email nhận được.

⭐⭐ BT6: Scheduled Report Flow

📝 Mức độ: Trung bình | Thời gian: 20 phút | Chương liên quan: Ch.7

Mục tiêu

Tạo scheduled flow gửi báo cáo tổng hợp hàng tuần từ SharePoint data.

Yêu cầu

  1. Trigger: Recurrence — hàng tuần vào thứ Hai 8:00 AM
  2. Get items từ SharePoint list (filter: modified trong 7 ngày qua)
  3. Tạo HTML table từ dữ liệu
  4. Tính tổng hợp: Count items, Sum amounts
  5. Compose email HTML đẹp với thống kê + bảng dữ liệu
  6. Gửi email tới team

Expressions cần dùng

Expressions
// Filter items trong 7 ngày qua
addDays(utcNow(), -7)

// Format ngày
formatDateTime(utcNow(), 'dd/MM/yyyy')

// Đếm items
length(body('Get_items')?['value'])

// Tạo filter query (OData)
Modified ge '@{addDays(utcNow(), -7)}'

⭐⭐⭐ BT7: Approval Workflow hoàn chỉnh

📝 Mức độ: Nâng cao | Thời gian: 30 phút | Chương liên quan: Ch.7

Mục tiêu

Xây dựng quy trình phê duyệt đơn nghỉ phép (Leave Request) hoàn chỉnh.

Yêu cầu

  1. SharePoint list "Leave Requests" với columns: Employee, Type (Annual/Sick/Personal), StartDate, EndDate, Days, Status, Comments
  2. Trigger: Khi có item mới (Status = "Pending")
  3. Condition: Nếu Days ≤ 3 → Single approval (Direct Manager). Nếu Days > 3 → Sequential approval (Manager → HR)
  4. Approval action với details: tên NV, loại nghỉ, ngày, lý do
  5. Nếu Approve → Update Status = "Approved", gửi email xác nhận
  6. Nếu Reject → Update Status = "Rejected", gửi email kèm comments
  7. Post notification lên Teams

Flow structure

Approval Flow Architecture
TRIGGER: When item created → Status = "Pending"
  ↓
CONDITION: Days <= 3 ?
  ├── YES: Start approval → Manager only
  └── NO:  Start approval → Manager
           ↓ (if approved)
           Start approval → HR Director
  ↓
CONDITION: Final Outcome = "Approve" ?
  ├── YES: Update Status → "Approved"
  │        Send approval email
  │        Post Teams: "✅ Leave approved"
  └── NO:  Update Status → "Rejected"
           Send rejection email (include comments)
           Post Teams: "❌ Leave rejected"

⭐⭐ BT8: Desktop Flow — Excel Data Processing

📝 Mức độ: Trung bình | Thời gian: 25 phút | Chương liên quan: Ch.8

Mục tiêu

Tạo desktop flow tự động xử lý file Excel: đọc dữ liệu, tính toán, và tạo báo cáo.

Yêu cầu

  1. Mở Power Automate for Desktop → New flow
  2. Tạo file Excel mẫu "SalesData.xlsx" với columns: Product, Quantity, UnitPrice, Date
  3. Flow actions:
    • Launch Excel → Open SalesData.xlsx
    • Read from Excel → Get all data into variable
    • Loop through each row → Tính Total = Quantity × UnitPrice
    • Write Total vào column mới
    • Tính Grand Total bằng variable
    • Ghi Grand Total vào cell cuối cùng
    • Save & Close Excel
  4. Hiển thị message box với kết quả: "Processed X rows. Grand Total: $Y"

Desktop Flow Structure

Desktop Flow Actions
1. Set variable: GrandTotal = 0
2. Launch Excel → "C:\Data\SalesData.xlsx"
3. Read from Excel worksheet → First row has headers
   → Store into: %ExcelData%
4. Get first free column/row → %FirstFreeRow%
5. FOR EACH %CurrentRow% IN %ExcelData%
   5.1. Set variable: Total = %CurrentRow['Quantity']% * %CurrentRow['UnitPrice']%
   5.2. Write to Excel → Column E, Current row → %Total%
   5.3. Set variable: GrandTotal = %GrandTotal% + %Total%
6. END FOR
7. Write to Excel → Cell E1 → "Total" (header)
8. Write to Excel → Cell F%FirstFreeRow% → %GrandTotal%
9. Save Excel
10. Close Excel
11. Display message → "Done! Grand Total: $" + %GrandTotal%

⭐⭐⭐ BT9: Power Apps + Power Automate Integration

📝 Mức độ: Nâng cao | Thời gian: 40 phút | Chương liên quan: Ch.2, Ch.7

Mục tiêu

Xây dựng app quản lý IT Support Tickets — Canvas App gọi Power Automate flow để gửi notifications.

Yêu cầu

  1. Power Automate (Instant Flow):
    • Trigger: PowerApps (V2)
    • Input parameters: TicketTitle (text), Category (text), Priority (text), Description (text), ReporterEmail (text)
    • Actions: Tạo item trên SharePoint "IT Tickets", gửi email tới IT team, post Teams message, gửi push notification
    • Return: Ticket ID và Link
  2. Canvas App:
    • Screen 1: Form nhập ticket (Title, Category dropdown, Priority radio, Description multiline)
    • Submit button gọi Power Automate flow
    • Hiện success message với Ticket ID
    • Screen 2: Gallery hiển thị "My Tickets" (filter theo User().Email)
    • Screen 3: Ticket Detail view

Power Fx — Gọi Flow

Button OnSelect
// Gọi Power Automate flow từ Canvas App
Set(varResult,
    CreateTicketFlow.Run(
        txtTitle.Text,
        ddCategory.Selected.Value,
        rdPriority.Selected.Value,
        txtDescription.Text,
        User().Email
    )
);

// Hiện kết quả
Notify(
    "✅ Ticket created! ID: " & varResult.ticketid,
    NotificationType.Success
);

// Reset form
Reset(txtTitle);
Reset(txtDescription);

// Chuyển sang screen danh sách
Navigate(MyTicketsScreen, ScreenTransition.Fade)

⭐⭐⭐⭐ BT10: Full Enterprise App — Employee Onboarding

📝 Mức độ: Chuyên sâu | Thời gian: 2+ giờ | Chương liên quan: Ch.2-10

Mục tiêu

Xây dựng hệ thống Onboarding nhân viên mới hoàn chỉnh — kết hợp tất cả kiến thức.

Yêu cầu

  1. Dataverse Tables:
    • New Employees (Name, Email, Department, StartDate, Manager, Status)
    • Onboarding Tasks (TaskName, Category, DueDate, AssignedTo, Status, NewEmployee lookup)
    • Equipment Requests (Item, Specs, Status, NewEmployee lookup)
  2. Model-driven App cho HR: Manage employees, track onboarding progress
  3. Business Process Flow: HR Review → IT Setup → Manager Welcome → Complete
  4. Canvas App cho Managers: Submit new hire request form
  5. Cloud Flows:
    • Khi new employee được tạo → Auto-create onboarding tasks
    • Khi task hoàn thành → Notify manager, update progress
    • Scheduled: Daily reminder cho overdue tasks
    • Approval: Equipment requests > $500 cần manager approval
  6. Dashboard: Onboarding status, tasks by department, completion rate
