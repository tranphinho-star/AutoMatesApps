⚡ Automated Flows

Automated flows được kích hoạt tự động bởi một sự kiện (trigger). Đây là loại flow phổ biến nhất.

Triggers phổ biến

ServiceTriggerMô tả
SharePointWhen an item is createdItem mới trong list
SharePointWhen an item is modifiedItem được cập nhật
OutlookWhen a new email arrivesEmail mới vào inbox
FormsWhen a response is submittedForm response mới
DataverseWhen a row is added/modifiedRecord thay đổi
TeamsWhen a keyword is mentionedTừ khóa xuất hiện
TwitterWhen a new tweet is postedTweet mới matching

Ví dụ: Auto-save Email Attachments

Automated Flow
TRIGGER: When a new email arrives (Outlook)
  → Filter: Has Attachments = Yes
  ↓
ACTION: Apply to each (attachment)
  ↓
  ACTION: Create file (OneDrive)
    - Folder: /Email Attachments/
    - File Name: attachment Name
    - File Content: attachment Content
  ↓
  ACTION: Send notification
    - "File saved: " + attachment Name

⏰ Scheduled Flows

Chạy theo lịch trình định kỳ: minutes, hours, daily, weekly, monthly.

📊

Weekly Reports

Tạo báo cáo tổng hợp hàng tuần từ SharePoint data và gửi email.

🧹

Data Cleanup

Xóa records cũ, archive data, clear temp files theo lịch.

🔄

Data Sync

Đồng bộ dữ liệu giữa hệ thống hàng đêm.

📧

Reminders

Gửi nhắc nhở hàng ngày cho tasks sắp đến hạn.

Scheduled Flow — Báo cáo tuần
TRIGGER: Recurrence
  → Frequency: Week | Day: Monday | Time: 8:00 AM
  ↓
ACTION: Get items (SharePoint - "Sales Data")
  → Filter: Modified last 7 days
  ↓
ACTION: Create HTML table
  → From: Filtered items
  ↓
ACTION: Send email (Outlook)
  → To: team@company.com
  → Subject: "📊 Weekly Sales Report - " + utcNow()
  → Body: HTML table content

👆 Instant Flows

Kích hoạt thủ công bằng button click — từ Power Apps, Power Automate mobile, Teams, hoặc SharePoint.

Trigger types

  • Manually trigger a flow: Button trong Power Automate
  • Power Apps: Gọi từ Canvas App button
  • For a selected item: Right-click context menu trên SharePoint item
  • For a selected row: Từ Dataverse record
  • From Teams: Adaptive card button, compose box
💡 Input parameters
Instant flows có thể nhận input từ user: text, number, date, file, email, yes/no. Cấu hình inputs trong trigger settings.

🔀 Conditions & Branching

Sử dụng Condition action để tạo logic If/Else:

Condition — Phân loại Priority
CONDITION: Priority equals "High"
  ├── IF YES:
  │   ├── Send email to Manager
  │   └── Create follow-up Task
  └── IF NO:
      └── Add to weekly digest list

Switch (Multi-branch)

Switch — Nhiều nhánh
SWITCH: Department
  ├── Case "IT":    → Assign to IT Team
  ├── Case "HR":    → Assign to HR Team
  ├── Case "Sales": → Assign to Sales Team
  └── Default:      → Assign to General

🔁 Loops & Apply to Each

Loop TypeMô tảVí dụ
Apply to eachLặp qua từng item trong arrayProcess each attachment, each list item
Do untilLặp cho đến khi điều kiện đúngPoll API until status = "Complete"
⚠️ Performance
  • Mặc định Apply to each chạy tuần tự. Bật Concurrency Control để chạy song song (max 50)
  • Do until có giới hạn 60 iterations và timeout 1 giờ mặc định

✅ Approval Workflows

Approval là tính năng mạnh mẽ nhất của Power Automate cho quy trình phê duyệt:

Loại Approval

👤

Approve/Reject - First

Gửi tới nhiều người, chỉ cần 1 người approve/reject.

👥

Approve/Reject - Everyone

Gửi tới nhiều người, tất cả phải approve.

📝

Custom Responses

Tùy chỉnh options: "Approve", "Reject", "Need More Info".

Approval Flow — Leave Request
TRIGGER: When a new item is created (SharePoint - "Leave Requests")
  ↓
ACTION: Start and wait for an approval
  - Type: Approve/Reject - First to respond
  - Title: "Leave Request from " + EmployeeName
  - Assigned to: ManagerEmail
  - Details: LeaveType + ": " + StartDate + " to " + EndDate
  ↓
CONDITION: Outcome equals "Approve"
  ├── YES: Update item → Status = "Approved"
  │        Send email → "✅ Your leave request was approved!"
  └── NO:  Update item → Status = "Rejected"
           Send email → "❌ Your leave request was rejected."

🛡️ Error Handling

  • Configure run after: Chọn action chạy khi action trước: succeeded, failed, skipped, timed out
  • Scope + Try/Catch: Nhóm actions trong Scope. Nếu fail, actions "run after" sẽ xử lý error
  • Retry policy: Tự động retry action khi fail (default: 4 retries)
  • Terminate: Dừng flow với status: Succeeded, Failed, hoặc Cancelled
Error Handling Pattern
SCOPE: "Try"
  ├── Action 1: Get data
  ├── Action 2: Process data
  └── Action 3: Save results

SCOPE: "Catch" (Run after: "Try" has failed)
  ├── Send error notification email
  └── Log error to SharePoint list
